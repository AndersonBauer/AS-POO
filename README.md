# AS-POO

Este projeto é uma API simples feita com Spring Boot, criada para lidar com dois tipos de informações: treinadores e jogadores. A ideia é basicamente ter treinadores cadastrados, cada um podendo ter vários jogadores associados a ele. A API permite cadastrar novos treinadores, editar seus dados, listar todos eles, buscar apenas um pelo ID e também excluir. O mesmo vale para os jogadores, que sempre precisam estar vinculados a um treinador existente. Quando um jogador é criado, ele recebe um conteúdo de texto e também o ID do treinador ao qual pertence. Depois disso, o projeto salva o jogador no banco de dados e mantém essa relação funcionando corretamente.

Para organizar melhor tudo que entra e sai da API, o projeto usa DTOs, que são classes criadas apenas para transportar dados de um jeito mais limpo. Do lado dos jogadores, existe um DTO que recebe as informações quando alguém envia um jogador novo e outro DTO que devolve as informações quando a API responde, incluindo também o nome do treinador desse jogador. Do lado dos treinadores, há um DTO que devolve o nome do treinador, seus títulos, prêmios e a lista de jogadores que pertencem a ele, cada jogador contendo apenas seu ID e o content. Dessa forma, as respostas da API ficam claras e sem excesso de dados.

A estrutura principal fica dividida em controladores, serviços, entidades e repositórios. Os controladores são a porta de entrada da API: eles recebem as requisições e devolvem respostas. Os serviços são onde a lógica acontece, como buscar um treinador, transformar objetos em DTO, salvar no banco e atualizar dados. As entidades são classes que representam os objetos reais, como o treinador e o jogador. Cada jogador tem uma ligação ManyToOne com um treinador, e cada treinador tem uma lista de jogadores, representando uma relação OneToMany. Os repositórios são responsáveis por conversar com o banco de dados sem que você precise escrever SQL manualmente.

Quando um treinador é buscado, a API devolve seus dados mais importantes junto com os jogadores vinculados a ele. Quando um jogador é buscado, a API devolve o seu conteúdo e também o nome do treinador responsável. Quando alguém altera um jogador, é possível mudar tanto o conteúdo quanto o treinador ao qual ele pertence. Por fim, sempre que um jogador ou treinador não é encontrado, o código lança um erro simples explicando isso.

O objetivo geral do projeto é ser uma API organizada, clara e fácil de entender, onde treinadores e jogadores se relacionam de forma simples e direta. Mesmo quem não tem muita experiência consegue compreender que o projeto serve para guardar essas informações, permitir buscas e atualizações e manter a relação entre treinador e jogador funcionando corretamente por meio do Spring Boot, JPA e das classes criadas para organizar os dados.
